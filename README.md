# 📊 SMT - Stock Market Trading Platform

Торговая платформа с поддержкой лимитных и рыночных ордеров, портфелем пользователя и историей сделок. Реализована на FastAPI + SQLAlchemy async + RabbitMQ.

---

## 🚀 Quick Start

1. В корне проекта создайте `.env` файл и заполните переменные:

* `POSTGRESQL_HOST` - хост БД
* `POSTGRESQL_PORT` - порт БД
* `POSTGRESQL_USER` - пользователь БД
* `POSTGRESQL_PASS` - пароль
* `POSTGRESQL_NAME` - название БД
* `POSTGRESQL_ECHO` - режим отладки БД (`True`/`False`, по умолчанию `False`)
* `SECRET_KEY` — секретный ключ для подписи JWT

2. Установите Python 3.13, Docker и Docker Compose
3. Соберите образ:
```bash
docker-compose build
```
4. Запустите:
```bash
docker-compose up -d
```

## 🧱 Стек

- **FastAPI** - REST API
- **SQLAlchemy 2.0** (async) - ORM
- **asyncpg** - драйвер PostgreSQL
- **Alembic** - миграции
- **aio-pika / RabbitMQ** - очередь заказов
- **pytest** / **pytest-asyncio** - тестирование
- **Docker** / **docker-compose** - инфраструктура

## 🗺️ Roadmap

Планируемые улучшения в порядке приоритета:

1. **E2E-тесты** - покрыть критичные пользовательские сценарии (регистрация, создание ордера, получение портфеля)
2. **Сервисный слой** - выделить бизнес-логику из хендлеров и репозиториев в отдельные сервисы
3. **Рефакторинг** - упорядочить модели, вынести общую логику, уменьшить связанность модулей, вернуть RabbitMQ в проект (ранее был выпилен по требованиям вуза)
4. **Unit и интеграционные тесты** - покрыть сервисный слой и репозитории, мокать внешние зависимости
5. **CI/CD** - автоматический запуск тестов, линтера и сборки Docker образа в GitHub Actions
6. **Middleware и мониторинг** - логирование запросов, health check, graceful shutdown
